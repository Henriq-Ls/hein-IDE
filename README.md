# O que torna a hein IDE diferente?

A maioria das IDEs possui editor, terminal e integração Git.

A hein IDE adiciona uma camada de entendimento do projeto.

Enquanto você desenvolve, a IDE indexa a estrutura do código, mantém conhecimento persistente, auxilia a Inteligência Artificial com contexto real e documenta automaticamente a evolução da aplicação.

Principais diferenciais:

* Inteligência Artificial integrada ao projeto
* Memória persistente entre sessões
* Núcleo de Conhecimento do projeto
* Documentação automática
* Arquitetura automática
* Diagnóstico estrutural via AST
* HEIN-PATCH para correções assistidas
* Git e GitHub integrados
* Suporte a Ollama e modelos locais
* Busca unificada do projeto

# Inteligência Artificial Contextual

<p align="center">
<img src="hein-ide-ai-panel.png" width="100%">
</p>

A IA da hein IDE não funciona como um chat isolado.

Ela entende a estrutura do projeto aberto, pode ler arquivos automaticamente, mantém histórico persistente entre sessões e utiliza contexto real durante as respostas.

Recursos:

* Leitura automática do projeto
* Injeção automática do arquivo aberto
* Memória persistente por projeto
* Streaming em tempo real
* Fallback automático entre provedores
* Suporte a OpenAI, OpenRouter e Ollama
* Entrada de voz via Whisper
* Revisão crítica automática através do sistema hein?
* HEIN-PATCH para aplicação de correções diretamente no editor

# Núcleo de Conhecimento

O Núcleo é uma camada exclusiva da hein IDE.

Seu objetivo é compreender a evolução do projeto e transformar informações dispersas em conhecimento reutilizável.

O Núcleo mantém:

* Histórico técnico
* Arquitetura da aplicação
* Estrutura dos módulos
* Diagnósticos automáticos
* Índice AST do projeto
* Grafo de dependências
* Alertas arquiteturais

Também pode gerar automaticamente:

* README.md
* DOCUMENTACAO.md
* ARQUITETURA.md
* Estrutura completa do projeto
